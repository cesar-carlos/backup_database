| Sintoma | Ação imediata |
|---|---|
| Senha de servidor remoto sumiu / cliente não consegue mais conectar | Reabrir diálogo de conexão e **re-salvar a senha**. Migração para vault DPAPI é transparente, mas se falhar a primeira leitura mantém o segredo no vault e zera só a coluna. |

### Trigger do appcast

⚠️ Importante: o workflow `update-appcast.yml` só roda em:
- `release: published` (novo release)
- `push: main` em `scripts/appcast_policy.json` / `sync_appcast_from_releases.py` / o próprio workflow
- `workflow_dispatch` (manual)

Isso significa que o ponteiro `min_publication_age_minutes` **não dispara
automaticamente quando atinge a idade**. Quando completar 60 min, o feed
**continuará sem 3.5.0** até algum trigger acima rodar.

Opções para liberar:
1. **Esperar próximo trigger natural** (próxima release ou edit no policy)
2. **Disparar manualmente**:
   ```powershell
   gh workflow run update-appcast.yml
   ```
3. **Reduzir o age e bumpar** (re-edit do policy serve de trigger):
   ```jsonc
   // simplesmente remove a entrada de age — vira "publicação imediata"
   "min_publication_age_minutes": {}
   ```

### Em logs do cliente (caso suporte reporte)

- [ ] `[auto-update] silencioso bloqueado: UAC ativo` — esperado em
      máquinas não-elevadas; conferir que **não** vem com `manual`
      no source.
- [ ] `[server_connections] Senha legada migrada para vault` —
      esperado uma vez por conexão pré-3.5.0.
- [ ] `[remote_schedules] tryResumeExecutionAfterReconnect já em
      execução` — esperado se UI estava aberta + reconexão.
- [ ] `[remote_schedules] Snapshot pré-restart restaurado` — esperado
      pós auto-update se havia backup remoto em execução.
- [ ] `Transferência ociosa por mais que 120s; abortando` — sinal real
      de servidor preso / rede caída. Investigar **antes** de culpar
      o cliente.

### Anti-sinais (algo deu errado)

- ❌ `[server_connections] Falha ao migrar senha legada` — vault DPAPI
     com problema. Operador precisa re-salvar a senha manualmente.
- ❌ `Hard ceiling de transferência atingido (360min)` — backup
     transferindo a < 100 KB/s; problema de rede ou disco do servidor.
- ❌ `tryResumeExecutionAfterReconnect` em loop (mais de 1 por minuto
     no mesmo `runId`) — guard de re-entrância falhou; abrir bug.

## Rollback rápido (incident response)

Se 3.5.0 quebrar produção, retirar do feed **sem apagar** a release:

```powershell
# 1. Edit scripts/appcast_policy.json
#    "blocked_versions": ["3.5.0"]
# 2. Commit + push main
git checkout main
# editar scripts/appcast_policy.json
git add scripts/appcast_policy.json
git commit -m "fix(appcast): bloquear 3.5.0 temporariamente (rollback)"
git push origin main
# 3. Workflow update-appcast roda automaticamente em ~30s
# 4. Confirmar: a entrada de 3.5.0 some do appcast.xml
git pull origin main
Select-String -Path appcast.xml -Pattern '3\.5\.0'
```

Clientes que **ainda não fizeram upgrade** param de receber a release.
Clientes que **já estão na 3.5.0** continuam — não há mecanismo de
downgrade automático (precisa rollback manual pelo operador via
`BackupDatabase-Setup-3.4.0.exe`).
