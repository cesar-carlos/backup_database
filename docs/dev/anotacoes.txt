Vamos analizar o funcionamento do projeto para criar um plano de uma implemetnacao completa..O objetivo é criar duas vertente deste mesmo projeto, um vai ser oque ja existe hoje, que vai ser chamado de Server e o novo sera o Cliente.. Os dois vai ser contruido dentro do mesmo projeto, o usuario durante o processo de instalacao que vai escolher se vai ser instalado o Server ou Client com suas respectivas implementacoes.. As funcionalidades do Server ja estao praticamente todas implementadas.. o Client tambem vai utilizar grande parte das implementacoes do Server, por isto sera feito dentro do mesmo projeto.
A finalizade co Client é ser um tipo de controle remoto do Server, executar backup e por protocolo Sockep TCP/IP transmitir para computador do client o arquivo de backup ja compactado.. para transmicao entre Server e Client vamos converter para binary no server e desconverter no Client. 
Devemos criar todo um controle de loguin, para cliente conectar em um servidor ou em varios servidores..um controle de cliente conectado no server, no client dos server online.. 
Oque vai ser controlado basicamente no Cliente, seria os agendamentos.. o client nao vai ter abilidade de criar agendamento, mais sim de controlalos, controle de trocar data de agendamento, executar remotamente, como agendamento vai ser processado todo no servividor e transmitido para maquina do cliente, como se fosse um driver remoto de armazenamento, temos que criar uma tela onde o cliente vai adicionar o controle, vai listar os servidores conecatado, (que ja foram conecados anteriormente por outra tela de LOGIN conserver), depois de ele escolher o server, vai listar os agendamentos que ele pode fazer o controle, dos agendamentos ele so pode trocar o tipo de backup, data de excusao, executar, comandos sql apos finalizar backup, a parte de armazenamento ele nao pode mexer porque este processo acorre no servidor. ele tmbm tem que escolher o destino do arquivo transmitido de backup apos terminar o backup, LOCAL, FTP, GoogleCloud, NexClound... o agendamento criado no servidor vai continuar no seu fluxo normal..
